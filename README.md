🎓 University API (ASP.NET Core + Docker + Monitoring)

Серверное приложение для управления университетской системой (студенты, курсы и т.д.) с поддержкой кэширования, мониторинга и контейнеризации.

📌 Описание проекта

Данное приложение представляет собой REST API, разработанное на платформе ASP.NET Core.

Приложение позволяет:
- Управлять сущностями (например: студенты, курсы)
- Выполнять CRUD-операции
- Использовать кэширование через Redis
- Хранить данные в PostgreSQL
- Отслеживать состояние приложения через метрики
- Визуализировать метрики в Grafana

🛠 Используемые технологии
- ASP.NET Core (.NET 8)
- Entity Framework Core
- PostgreSQL
- Redis (кэширование)
- Docker / Docker Compose
- Prometheus (сбор метрик)
- Grafana (визуализация)
- REST API
- Dependency Injection

🏗 Архитектура проекта

Проект построен по принципам чистой архитектуры и разделения ответственности.

🚀 Реализованные функции

✔ CRUD операции (создание, чтение, обновление, удаление)

✔ Подключение PostgreSQL

✔ Кэширование через Redis

✔ Docker-контейнеризация

✔ Мониторинг через Prometheus

✔ Визуализация метрик в Grafana

✔ Endpoint /metrics

✔ Обработка ошибок

🐳 Docker-инфраструктура

Проект запускается через Docker Compose и включает:

- university_api — ASP.NET Core API
- postgres_db — база данных PostgreSQL
- redis_cache — кэш Redis
- prometheus — сбор метрик
- grafana — визуализация

📊 Мониторинг системы
Prometheus
Собирает метрики с API:
- HTTP запросы
- Время ответа
- Ошибки

Endpoint:
-http://localhost:5000/metrics
- Grafana
Используется для построения дашбордов.

Доступ:
- http://localhost:3000
Логин/пароль:
- admin / admin

📈 Примеры метрик

Минимум 3 метрики:
- Количество HTTP-запросов
- Время обработки запросов
- Количество ошибок

🖥 Пользовательский интерфейс

В рамках проекта реализован веб-интерфейс (или используется Swagger UI):
- http://localhost:5000/swagger

Позволяет:
- Тестировать API
- Отправлять запросы
- Просматривать ответы

⚙ Запуск проекта

🔹 1. Сборка и запуск
- docker-compose up --build

🔹 2. Проверка сервисов
- API → http://localhost:5000
- Swagger → http://localhost:5000/swagger
- Metrics → http://localhost:5000/metrics
- Grafana → http://localhost:3000
- Prometheus → http://localhost:9090

🔎 Принцип работы приложения

- API принимает HTTP-запрос
- Обрабатывает через контроллер
- Обращается к сервисам
- При необходимости использует Redis (кэш)
- Сохраняет/читает данные из PostgreSQL
- Метрики отправляются в Prometheus
- Grafana отображает данные в виде графиков
